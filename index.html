Modbus Export Processor

Private repository — available on request

A Python-based data processing tool that reads Modbus register definitions from structured Excel files and converts them into a clean, production-ready configuration for Telegraf agent. The program filters alarms and warnings, applies custom byte-order and formatting logic, and automatically generates Modbus input and holding register sections that can be appended directly to an existing telegraf.conf. Designed to streamline industrial data acquisition workflows and reduce manual configuration errors.

Tech: Python, Pandas

TIG Stack with Local + Cloud Telegraf Pipelines

Private repository — available on request

A fully containerized monitoring and data ingestion platform based on the TIG stack (Telegraf, InfluxDB, Grafana), designed for environments with unreliable connectivity such as marine networks. The solution runs two Telegraf instances locally: one writes directly into a local InfluxDB for on-site monitoring, while the second forwards the same measurements to a separate cloud InfluxDB instance. Kafka is added as an intermediary between the local and cloud path to buffer telemetry and prevent data loss during intermittent connections or outages, enabling reliable store-and-forward behavior across unstable networks. The project includes automated provisioning of InfluxDB data sources, environment-driven Grafana dashboard templating, secure credential handling, and cross-platform deployment scripts.

Tech: Docker, Telegraf, InfluxDB, Grafana, Kafka, Linux, Cloud - Digital Ocean, Bash/Shell scripts
